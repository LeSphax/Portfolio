OpenAI Gym


Technologies : Python, Tensorflow

I started getting really interested in reinforcement learning in January 2018 when I discovered Unity MLAgents framework. I had fiddled a bit with tensorflow before but being able to use ML on video games made it really appealing.

But after trying out MLAgents, I realized that to use it effectively I would need to first understand actor critic algorithms and particularly ppo.

So using as a reference the ppo paper, David Silver's course on RL and openAI baselines I reimplemented the ppo algorithm and tested it on two gym environments: CartPole and Atari Breakout.

I ended up getting similar results to the openAI ppo baseline on these two environments.

Afterwards, I also added curiosity based rewards based on this work and tested it on an environment where an agent must click buttons appearing at random positions

Unity ML Agents


Technologies : Python, Tensorflow, Unity

Now that I am familiar with reinforcement learning and the ppo algorithm I am using the Unity ML agents framework again.

The objective is to teach agents to play my game Slideball. First I want to see how far I can get using the algorithms built in MLAgents (ppo with curiosity, curriculum learning and behavioral cloning). In the future, I might also use this framework to implement and test other algorithms.

At the moment (December 2018), I have successfully taught my agents to pick up a ball and throw it into goals placed at random positions. I am working on the next steps which is to make them play against each other. For that, I am adding new features to the framework to allow agents to play againt past versions of themselves. As well as an elo rating system to be able to measure the progression of the agents skill level.

Perspective Shift


Technologies : Unity with C#

This is a puzzle game that progressively introduce new mechanics to the player that they must understand to progress further. At the moment there are only puzzles and no real structure to navigate them.

The original idea for the 2D game mechanics came from a friend, he also created the 3D models. Other than that, I built everything in this demo.ø

Implementing the undo feature while having animations between the different board states was quite challenging and I had to build a complex hierarchy of objects and events. Another challenge was to switch between the 2D and 3D modes.
It was also my first real try at level design.

I stopped working on it in September 2016 and decided to focus on SlideBall instead.

Technologies : Unity with C#

Tools : GIMP, Blender, Maya, Audacity

I really liked the mod Hover on Starcraft II and I am trying to create a similar game by myself.

For the networking side, I started to use Photon Unity Network but I then switched to WebRTC to get a better latency. This means I had to recreate several functionnalities like RPCs and distributing packets to the different entities and I am hosting my own signaling server that handles the rooms creation.

This is the game I am currently working on (May 2017). The technical side starts to work reliably well when playing with 8 people so I am starting to experiment more with the gameplay.

FlyHigher


Technologies : Unity with C#, XML
Tools : GIMP

Working with 3 other students, I made a game to teach aeronautics to children from 6 to 10 years old and produced a small paper containing recommandation for creating educational entertainment.

After learning how to use Unity, I created two of the 9 small games, the save management and translation modules and half of the game UI and navigation. I ported the game from android to WebGL.

Designing, developing and testing the game took about 100 hours of work from each of us. I also organised and conducted user tests with primary school children.
